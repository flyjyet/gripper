# 2026-05-20 控制功能补齐记录

## 本次目标

在电机通信、使能、失能和 Bring-up 点动链路确认正常后，补齐夹爪控制器中其余核心功能，使 PC UI 可以按真实硬件调试流程逐步测试：

1. PreSelfCheck。
2. 回零。
3. 行程学习。
4. 运动健康检查。
5. 目标力夹紧。
6. 释放。
7. 主动停止和动作后自动失能。

## 核心改动

### 1. 行程参考系

新增控制器内部行程参考逻辑：

- 未回零前使用临时中位参考，仅用于小范围预自检探测。
- 回零成功后，将当前电机反馈位置记录为开口零点。
- 后续行程命令统一通过“电机零点 + 丝杆导程 + 方向符号”换算。

这样避免把达妙电机绝对编码器的零位误认为夹爪开口零位。

### 2. 通用运动等待循环

新增统一的运动等待逻辑：

- 发出限流、限速、限行程后的运动命令。
- 周期读取电机反馈。
- 判断目标行程、接触、电流代理、堵转和超时。
- 动作结束后统一失能，避免丝杆螺母持续预紧导致抱死。

### 3. PreSelfCheck

PreSelfCheck 现在会：

- 按配置逐步扫描速度和电流。
- 尝试开闭两个方向。
- 记录可动样本、摩擦代理样本和往返样本。
- 生成保守结构参数。

注意：当前结构参数质量仍为 `LowConfidence`。它适合调试流程打通和保守控制，不等价于样机标定后的最终结构模型。

### 4. 回零

回零现在会：

- 以低速、低电流向打开方向运动。
- 通过反馈确认产生可测运动。
- 成功后记录电机当前位置为开口零点。
- 完成后自动失能。

### 5. 行程学习

行程学习现在会：

- 从开口零点向闭合方向运动到理论闭合端。
- 记录实际闭合侧反馈位置。
- 生成软件开限位和软件闭限位。
- 建立软件限位后回退到软件开限位附近。
- 完成后自动失能。

### 6. 运动健康检查

运动健康检查现在会：

- 在软件限位内按配置速度进行闭合和打开往返。
- 记录速度跟踪、温度、行程范围等样本。
- 通过后进入 `Ready`。
- 完成后自动失能。

### 7. 夹紧和释放

目标力夹紧现在会：

- 使用目标力到电机电流/力矩的一阶映射。
- 按配置速度闭合。
- 目标电流代理达到或接触检测达到时停止。
- 正常完成后自动失能。
- 如果只到达软件闭合限位、但没有接触或电流代理确认，视为未成功夹紧。

释放现在会：

- 按释放距离和释放速度向打开方向运动。
- 到达目标后清除估算夹紧力和接触标志。
- 完成后自动失能。

## UI 更新

Web UI 普通控制区补充：

- 目标夹紧力输入。
- 夹紧速度输入。
- 按后端状态禁用不可用按钮。

管理员恢复区仍保留门禁显示，但高电流恢复、低能量方向试探和参数写回仍标记为未实现，不会下发电机命令。

## 实机测试顺序建议

1. `connect`
2. `enter_bringup`
3. `can_probe`
4. `feedback`
5. Velocity 正反点动，确认方向和电流。
6. `exit_bringup`
7. `selfcheck`
8. `home`
9. `learn`
10. `health`
11. 小目标力夹紧，例如 `20 N`。
12. `release`

首次结构安装后建议从低速度、低目标力开始，不直接测试 200 N。

## 仍需后续标定/验证

- 目标力到电机电流/力矩的映射仍是一阶配置模型，需要样机标定。
- 接触检测和堵转阈值需要真实线缆、不同直径和不同速度下验证。
- PreSelfCheck 的摩擦参数当前来自电机反馈代理，质量为低置信。
- 管理员高权限恢复功能尚未实现后端安全策略，不能用于实机恢复。

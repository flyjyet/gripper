# 夹爪控制系统设计

日期：2026-05-14

## 1. 设计目标

本控制系统面向当前机器人夹爪机构：直流伺服电机 + 梯形丝杆 + 连杆夹爪。系统除电机自身反馈外，没有额外传感器；电机提供单圈绝对值双编码器，以及电流、力矩、速度、位置、温度等反馈。

控制设计目标：

- 全行程限流、限速、软限位兜底，避免过大电流或速度损伤结构。
- 支持低速低电流堵转靠零，建立螺母行程零点。
- 支持目标夹紧力夹紧和指定恒定夹紧速度夹紧。
- 全行程预防冲击，不能假设 `0-12 mm` 区间一定空载。
- 主动堵转检测，主动停止，避免丝杆螺母卡死。
- 夹紧完成后电机失能，不持续输出力矩，防止螺母抱死。
- 打开前具备低电流反向解锁和分级防卡死恢复。
- 初始化自检时从小力矩开始识别机构内部摩擦力矩，并用于后续阈值补偿。
- 软件模块职责清晰，硬件抽象通过接口、回调或子类实现。

## 2. 已知机构约束

| 项目 | 数值或结论 |
|---|---:|
| 螺母行程 | 0-16 mm |
| 线缆直径范围 | 14-28 mm |
| 其他物体夹持 | 可能出现在 0-16 mm 任意行程，尤其 0-12 mm 不能视为空载 |
| 丝杆导程 | 2 mm |
| 电机额定扭矩 | 3.5 N*m |
| 电机峰值扭矩 | 12.5 N*m |
| 目标夹紧力 | 默认单侧 200 N，样机初测建议从 150 N 开始 |
| 结构风险 | 3 mm 销轴动态裕量不足，必须严格限速限流 |
| 夹紧保持 | 夹紧后电机失能，依赖梯形丝杆自锁 |

机构研究结论显示：夹爪运动和螺母行程为非线性关系，闭合末端速度比和加速度几何项陡增。因此控制策略必须优先降低接触冲击和防止丝杆螺母抱死。

## 3. 总体控制原则

### 3.1 全行程保护

从 `s=0` 到 `s=16 mm`，任何位置都可能接触物体，因此所有闭合运动都必须启用：

- 最大速度限制。
- 最大电流/力矩限制。
- 软限位保护。
- 接触检测。
- 堵转检测。
- 温度和反馈异常检测。

`12 mm` 只作为 14-28 mm 线缆接触区的参考位置，不作为“无保护快速接近”的边界。

### 3.2 防抱死优先

当前系统经常出现丝杆螺母卡死，控制策略必须避免以下行为：

- 高速接触后继续给位置命令硬顶。
- 到达目标力后仍持续输出力矩。
- 夹紧后直接失能而不释放轴向预紧尖峰。
- 打开时直接大电流反拉卡死螺母。
- 异常堵转后继续提高电流尝试动作。

### 3.3 分阶段夹紧

推荐夹紧流程：

```text
自检/回零 -> 全程受保护闭合 -> 接触检测 -> 速度置零/降速 -> 限流建立夹紧力 -> 卸载预紧 -> 失能保持
```

## 4. 状态机设计

### 4.1 状态列表

| 状态 | 说明 |
|---|---|
| `Disabled` | 电机失能，默认安全状态 |
| `PowerOnSelfCheck` | 上电自检，检查反馈、温度、通信、故障字 |
| `HomingOpenStop` | 低速低电流向打开方向靠零 |
| `FrictionIdentify` | 小力矩低速往返，识别空载摩擦 |
| `Ready` | 已完成初始化，可接收命令 |
| `GuardedClosing` | 全行程受保护闭合，接触检测全程有效 |
| `ContactDetected` | 检测到接触，准备切换夹紧建立 |
| `ForceBuild` | 限流/限扭建立目标夹紧力 |
| `UnloadBeforeDisable` | 夹紧后释放丝杆轴向预紧尖峰 |
| `ClampDoneDisabled` | 夹紧完成，电机失能，依赖丝杆自锁保持 |
| `UnlockBeforeOpen` | 打开前低电流反向解锁 |
| `Opening` | 受保护打开 |
| `AntiJamRelease` | 异常卡死后的分级解锁 |
| `FaultStop` | 故障停机，等待复位或人工处理 |

### 4.2 正常初始化流程

```text
Disabled
  -> PowerOnSelfCheck
  -> HomingOpenStop
  -> FrictionIdentify
  -> Ready
```

#### PowerOnSelfCheck

检查项：

- 电机通信在线。
- 编码器反馈有效。
- 电流、速度、位置、温度反馈时间戳有效。
- 驱动器无严重故障。
- 当前温度低于启动阈值。

若失败，进入 `FaultStop`。

#### HomingOpenStop

目的：不依赖掉电保存多圈位置，每次上电建立螺母零点。

策略：

- 以打开方向低速运动。
- 使用回零专用低电流限制。
- 若检测到低速低电流堵转，判定到达打开端机械零位。
- 设当前位置为 `s=0`。
- 回零过程不允许高电流硬顶。

#### FrictionIdentify

目的：识别机构内部摩擦，用于接触和堵转阈值补偿。

策略：

- 在零点附近安全小行程内低速闭合/打开。
- 电流上限低于夹紧电流。
- 记录开向和合向空载电流/力矩。
- 得到：

```text
I_friction_close
I_friction_open
T_friction_close
T_friction_open
```

若摩擦显著高于历史正常范围，进入故障或降额模式。

### 4.3 正常夹紧流程

```text
Ready
  -> GuardedClosing
  -> ContactDetected
  -> ForceBuild
  -> UnloadBeforeDisable
  -> ClampDoneDisabled
```

#### GuardedClosing

输入：

- 目标夹紧力。
- 指定夹紧速度。
- 最大允许电流/力矩。
- 超时时间。

控制：

- 速度模式优先，保持指定恒定夹紧速度。
- 所有速度命令经 `SafetyLimiter` 裁剪。
- 所有电流/力矩命令经 `SafetyLimiter` 裁剪。
- 接触检测在全行程有效。
- 若未接触但到达软限位 `s=16 mm`，停止并报“未检测到夹持对象/到达闭合软限位”。

#### ContactDetected

检测到接触后：

- 立即将速度命令降为 0，或降至极低速度。
- 冻结继续向闭合方向的大位置命令。
- 记录接触行程 `s_contact`。
- 根据接触行程估计夹持对象区间，用于选取标定表。

#### ForceBuild

控制目标：建立目标夹紧力，而不是继续追位置。

策略：

- 使用标定表将目标夹紧力映射为目标电流/扭矩。
- 叠加摩擦补偿：

```text
I_target = I_force_table + I_friction_close + I_margin
```

- 采用斜坡上升，避免电流阶跃。
- 达到目标后保持短暂稳定确认。
- 稳定后进入 `UnloadBeforeDisable`。

#### UnloadBeforeDisable

目的：释放丝杆螺母过高预紧，降低夹紧后卡死风险。

策略：

- 停止闭合方向输出。
- 执行极小反向卸载动作，或短时反向小力矩脉冲。
- 卸载幅度由样机标定，必须不破坏夹持可靠性。
- 确认速度接近零后失能。

初始建议：

```text
卸载位移：0.02-0.10 mm 试验标定
或反向电流：目标夹紧电流的 5%-15%
持续时间：20-100 ms
```

实际值必须通过样机验证。

### 4.4 打开流程

```text
ClampDoneDisabled
  -> UnlockBeforeOpen
  -> Opening
  -> Ready
```

#### UnlockBeforeOpen

目的：防止夹紧后丝杆螺母抱死，打开时不能直接大电流反拉。

策略：

- 使能电机。
- 先施加低电流打开方向命令。
- 若位置开始变化，进入 `Opening`。
- 若位置不动且电流上升，进入 `AntiJamRelease`。

#### Opening

策略：

- 速度受限。
- 电流受限。
- 到达 `s=0` 附近后低速靠零或停止。
- 若发生异常堵转，进入 `AntiJamRelease` 或 `FaultStop`。

### 4.5 防卡死恢复

`AntiJamRelease` 用于处理螺母抱死、异常堵转或打开失败。

分级策略：

| 等级 | 动作 | 判定 |
|---|---|---|
| 1 | 低电流打开方向持续尝试 | 位置变化则恢复 |
| 2 | 短时打开方向脉冲，间隔等待 | 位置变化则恢复 |
| 3 | 小幅闭合再打开，释放静摩擦 | 位置变化则恢复 |
| 4 | 停机报故障 | 禁止继续加大电流硬拉 |

每一级都必须限制最大电流、持续时间和尝试次数。

## 5. 接触、堵转和故障检测

### 5.1 检测信号

可用反馈：

- 电机位置。
- 电机速度。
- 电机电流。
- 电机估算力矩。
- 电机温度。
- 驱动器故障状态。

派生量：

```text
position_error = position_cmd - position_feedback
speed_error = speed_cmd - speed_feedback
current_over_friction = current - I_friction_direction
position_delta_in_window
```

### 5.2 接触检测

接触检测在闭合全行程有效。

建议组合判据：

```text
closing_command_active == true
current_over_friction > I_contact_delta
abs(speed_feedback) < speed_contact_threshold
speed_error > speed_error_threshold
duration > contact_debounce_ms
```

任意单一信号不应作为唯一判据，避免摩擦变化或反馈噪声误判。

### 5.3 堵转分类

| 类型 | 场景 | 处理 |
|---|---|---|
| `ExpectedHomingStall` | 回零碰到打开端 | 设零点，不报故障 |
| `ContactStall` | 闭合接触物体 | 切换 `ForceBuild` |
| `AbnormalJam` | 非预期堵转或卡死 | 停止，进入 `AntiJamRelease` 或 `FaultStop` |

### 5.4 主动堵转检测

主动堵转判据：

```text
abs(speed_cmd) > speed_cmd_min
abs(speed_feedback) < speed_stall_threshold
current_over_friction > I_stall_delta
position_delta_in_window < position_delta_min
duration > stall_debounce_ms
```

若在 `GuardedClosing` 中触发，可优先判断为接触；若在 `Opening` 或非预期状态触发，应判定为异常卡死。

### 5.5 温度和电流保护

温度保护分两级：

- 温度预警：降低最大速度和最大电流。
- 温度故障：停止并失能。

电流保护分三类：

- 回零电流上限。
- 正常夹紧电流上限。
- 瞬时绝对电流上限。

任何状态不得绕过瞬时绝对电流上限。

## 6. 安全限幅

所有命令统一经过 `SafetyLimiter`。

### 6.1 位置限幅

```text
0 <= s_cmd <= 16.0 mm
```

软限位必须早于机械限位生效。

### 6.2 速度限幅

建议配置：

| 参数 | 说明 |
|---|---|
| `v_close_unknown_max` | 未知物体闭合最大速度 |
| `v_close_known_max` | 已知线缆模式闭合最大速度 |
| `v_contact_search_max` | 接触搜索最大速度 |
| `v_open_max` | 打开最大速度 |
| `v_homing_max` | 回零最大速度 |
| `v_unlock_max` | 解锁最大速度 |

默认优先级：防卡死和防冲击高于夹紧速度。

### 6.3 电流/力矩限幅

建议配置：

| 参数 | 说明 |
|---|---|
| `I_abs_max` | 绝对最大电流 |
| `I_homing_max` | 回零最大电流 |
| `I_selfcheck_max` | 自检最大电流 |
| `I_close_guard_max` | 受保护闭合最大电流 |
| `I_force_target_max` | 目标夹紧最大电流 |
| `I_unlock_max` | 解锁最大电流 |
| `I_anti_jam_max` | 防卡死恢复最大电流 |

### 6.4 加速度和斜坡限制

为降低冲击，速度和电流不允许阶跃：

```text
|dv/dt| <= a_cmd_max
|dI/dt| <= current_slew_rate_max
```

接触后电流建立必须采用斜坡。

## 7. 夹紧力控制

### 7.1 标定表优先

无外部力传感器，夹紧力不能实时直接测量。控制系统采用标定表：

```text
target_force_per_side_N + contact_position_s + friction_params
    -> target_current_or_torque
```

标定表来源：

- 理论计算作为初值。
- 样机夹紧力测试修正。
- 按线径或接触行程分段。

### 7.2 未知物体模式

由于 0-12 mm 区间可能夹其他物体，必须支持未知物体模式：

- 不依赖线径输入。
- 根据实际接触行程选择参数区间。
- 使用更保守速度和电流斜坡。
- 目标力仍按指定值执行，但受最大安全电流限制。

### 7.3 指定恒定夹紧速度

闭合阶段按用户指定速度运行，但实际命令为：

```text
v_cmd = min(v_user, v_state_limit, v_safety_limit)
```

接触后不再保持原闭合速度，应进入：

```text
速度置零/极低速 -> 电流斜坡建立目标力
```

这能避免接触后继续积累丝杆预紧。

## 8. 软件架构

### 8.1 模块关系

```text
Application
  -> GripperController
      -> GripperStateMachine
      -> SafetyLimiter
      -> ContactAndJamDetector
      -> FrictionIdentifier
      -> ForceCommandMapper
      -> AntiJamManager
      -> MotorInterface
```

原则：

- 状态机不直接操作硬件。
- 所有硬件访问通过 `MotorInterface`。
- 所有命令输出通过 `SafetyLimiter`。
- 检测模块只输出事件，不直接切状态。
- 标定表和参数独立存储，支持后续测试更新。

### 8.2 硬件抽象接口

建议 C++ 接口：

```cpp
enum class MotorMode {
    Disabled,
    Position,
    Velocity,
    Current,
    Torque
};

struct MotorFeedback {
    double timestamp_s;
    double position_motor_rev;
    double velocity_motor_rev_s;
    double current_A;
    double torque_Nm;
    double temperature_C;
    uint32_t fault_bits;
    bool encoder_ok;
    bool drive_enabled;
};

struct MotorCommand {
    MotorMode mode;
    double target_position_rev;
    double target_velocity_rev_s;
    double target_current_A;
    double target_torque_Nm;
};

class MotorInterface {
public:
    virtual ~MotorInterface() = default;
    virtual bool enable() = 0;
    virtual bool disable() = 0;
    virtual bool resetFault() = 0;
    virtual bool sendCommand(const MotorCommand& command) = 0;
    virtual bool readFeedback(MotorFeedback& feedback) = 0;
};
```

具体电机硬件通过子类实现，例如：

```cpp
class CanServoMotor : public MotorInterface {};
class EthercatServoMotor : public MotorInterface {};
class SimulatedMotor : public MotorInterface {};
```

也可以在硬件层使用回调：

```cpp
using FeedbackCallback = std::function<bool(MotorFeedback&)>;
using CommandCallback = std::function<bool(const MotorCommand&)>;
```

## 9. 参数初值建议

以下为控制设计初值，必须通过样机测试修正。

### 9.1 速度策略

| 模式 | 建议 |
|---|---|
| 未知物体闭合 | 使用低速，优先防冲击 |
| 已知线缆闭合 | 允许分段速度，但全程接触检测和限流 |
| 接触后 | 速度置零或极低速 |
| 打开前 | 先低速低电流解锁 |
| 回零 | 低速低电流 |

### 9.2 力和电流策略

| 项目 | 建议 |
|---|---|
| 样机初测目标力 | 单侧 150 N |
| 目标验证力 | 单侧 200 N |
| 电流建立 | 斜坡上升 |
| 夹紧完成 | 卸载预紧后失能 |
| 异常卡死 | 禁止高电流硬拉 |

### 9.3 夹紧力目标

200 N 单侧目标夹紧力在驱动能力上可实现，但 3 mm 销轴动态裕量不足。控制参数应保证：

- 接触速度足够低。
- 电流斜坡足够缓。
- 不持续顶紧。
- 防卡死释放有效。

## 10. 测试验证

### 10.1 功能测试

- 上电自检是否能识别反馈异常、温度异常、驱动故障。
- 低力回零是否能重复建立 `s=0`。
- 摩擦识别是否能得到稳定开/合方向摩擦电流。
- 正常打开/闭合是否满足限速限流。

### 10.2 全行程障碍物测试

在不同位置放置测试物：

```text
s = 2、5、8、11、13、15 mm
```

验证：

- 均能检测接触。
- 均不会高速硬顶。
- 均能限流停止或进入目标力夹紧。
- 均能夹紧后失能并正常打开。

### 10.3 夹紧力测试

测试对象：

- 14、20、28 mm 线缆。
- 0-12 mm 区间的典型其他物体。

测试目标：

- 150 N。
- 200 N。

记录：

- 接触行程。
- 峰值电流。
- 稳态电流。
- 实际夹紧力。
- 打开是否卡死。

### 10.4 防冲击测试

对不同闭合速度测试：

```text
0.1、0.2、0.5、1.0、2.0 mm/s
```

记录：

- 峰值夹紧力。
- 冲击系数 `K_impact`。
- 峰值电流。
- 稳定时间。
- 是否出现丝杆卡死。

建议：

```text
K_impact <= 1.5
若 3 mm 销轴且 200 N 工况 K_impact > 1.2，应停止该销轴方案。
```

### 10.5 防抱死测试

测试项：

- 夹紧后立即失能，再打开。
- 夹紧后等待 10 s、60 s、10 min，再打开。
- 目标力 150 N、200 N 分别测试。
- 不同卸载位移/反向脉冲参数对打开成功率的影响。

判定：

- 打开不需要高电流硬拉。
- 解锁阶段电流不超过限制。
- 丝杆螺母无卡死。
- 多次循环后无卡死趋势增加。

### 10.6 异常堵转测试

场景：

- 闭合中人为阻挡。
- 打开中人为阻挡。
- 回零碰到机械端点。
- 夹紧后人为增加预紧。

验证：

- 能区分预期回零堵转、正常接触和异常卡死。
- 异常卡死不继续加大电流。
- 能进入 `AntiJamRelease` 或 `FaultStop`。

## 11. 归档结论

控制系统必须从当前结构风险出发设计，而不是只实现电机位置控制。关键结论：

1. 全行程都可能接触物体，`0-12 mm` 不能视为空载快速区。
2. 限流、限速、接触检测和堵转检测必须全程有效。
3. 目标力夹紧应在接触后用电流/扭矩斜坡建立，不继续追位置。
4. 夹紧完成后应卸载预紧并失能，避免持续输出力矩导致丝杆螺母抱死。
5. 打开前必须低电流解锁，异常卡死时禁止高电流硬拉。
6. 初始化摩擦识别是后续接触检测和堵转检测的基础。
7. 硬件必须通过抽象接口接入，控制状态机不直接依赖具体电机驱动实现。

后续实现优先级：

```text
1. MotorInterface 和反馈数据结构
2. SafetyLimiter 全局限速限流
3. 低力回零和摩擦识别
4. ContactAndJamDetector
5. GuardedClosing + ForceBuild
6. UnloadBeforeDisable + UnlockBeforeOpen + AntiJamRelease
7. 标定表和测试工具
```

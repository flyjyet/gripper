# 夹爪控制系统联调归档

日期：2026-05-15

## 1. 当前结论

当前 PC 测试程序已经完成基础硬件联调，夹爪控制链路可以跑通。已经验证的范围包括：

- PC UI 可以连接 `DM-USB2FDCAN_Dual`。
- 通过达妙 `DM_DeviceSDK v1.1` 可以打开设备、配置 FDCAN 通道并收发电机帧。
- 电机反馈可以被解析，UI 可以显示位置、速度、电流、力矩、温度、使能状态。
- `Enable`、`Home low-current`、`Clamp force`、`Open`、`Stop + Disable` 等基础操作已经进入可测试状态。
- 低速、低电流、相对小步位置目标的控制方式初步解决了“使能后无运动”和“命令目标等于当前位置”的问题。

当前程序仍属于样机调试和需求澄清阶段，不应直接作为最终产品控制软件使用。后续需要重构模块边界、补齐标定、测试和异常恢复策略。

## 2. 机械和控制前提

已确认的机构和控制前提：

| 项目 | 当前值或结论 |
|---|---|
| 执行机构 | 直流伺服电机 + 梯形丝杆 + 连杆夹爪 |
| 丝杆导程 | 2 mm |
| 有效螺母行程 | 约 16 mm |
| 线缆直径范围 | 14-28 mm |
| 其他夹持对象 | 可能出现在 0-12 mm 行程区间 |
| 控制反馈 | 电机自身位置、速度、电流、力矩、温度、故障状态 |
| 额外传感器 | 无 |
| 夹紧后保持 | 目标是卸载后电机失能，依靠丝杆自锁保持 |

关键控制原则：

- 全行程都要限速限流，不能把 0-12 mm 当作无保护空程。
- 夹紧接触瞬间必须避免高速和大位置目标硬顶。
- 夹紧完成后必须卸载并失能，降低丝杆螺母抱死风险。
- 打开前需要低电流解锁，不能一开始就大电流反拉。
- 无力传感器条件下，目标夹紧力只能通过电流、力矩、机构模型和标定估算。

## 5. 当前实现要点

### 5.1 达妙控制模式

当前 UI 后端采用达妙协议：

| 功能 | 协议行为 |
|---|---|
| 控制模式切换 | 写寄存器 `CTRL_MODE=0x0A` |
| 位置力控模式 | 模式值 `4`，控制帧 ID 为 `0x300 + motor_id` |
| 速度限幅 | POS_FORCE 帧中的速度限幅，单位 rad/s，乘 100 转为 uint16 |
| 电流限幅 | POS_FORCE 帧中的归一化电流，`current_limit_A / Max phase A` |
| MIT 力矩命令 | 用于当前 `ForceBuild` 阶段的电流/力矩建立 |
| 使能/失能 | 特殊帧 `ff ff ff ff ff ff ff fc/fd` |

重要修正：

- 后端初始 `current_mode` 不再假定为 `POS_FORCE`，首次命令会强制写控制模式。
- `Enable` 前会先切到 `POS_FORCE`，再发送模式相关的使能帧。
- 未归零前不再把第一帧反馈自动设置为软件零点。
- 未归零时的运动命令使用当前位置附近的小步相对目标，避免大位置目标和当前位置保持。
- 默认启用 CANFD/BRS 电机控制帧，以匹配当前设备 FDCAN 1M/5M 配置。

### 5.2 回零

`Home low-current` 当前策略：

1. 使能电机。
2. 切换到 `POS_FORCE`。
3. 向打开方向发送低速、低电流的小步相对目标。
4. 检测低速、低电流堵转。
5. 满足去抖条件后，将当前位置设置为软件零点。
6. 发送停止命令，进入 `Ready`。

如果在未归零状态下持续没有运动也没有电流，UI 会进入故障并提示检查：

- `Motor frames CANFD`
- 控制模式切换
- `Stroke sign`
- 电流限幅
- CAN ID 和通道配置

### 5.3 夹紧

`Clamp force` 当前流程：

```text
Ready
  -> GuardedClosing
  -> ForceBuild
  -> UnloadBeforeDisable
  -> ClampDoneDisabled
```

当前闭合阶段使用限速限流位置力控，接触检测使用：

```text
abs(current_A) - friction_close_A >= contact_delta
abs(velocity_mm_s) <= stall_speed_threshold
duration >= contact_debounce
```

当前目标夹紧力到电流的换算仍是占位模型：

```text
target_current_A = min(max_current_A, 0.30 + target_force_N / 200.0 * 0.90)
```

这只能用于联调，不是最终力控标定结果。

### 5.4 夹紧后卸载和失能

夹紧建立后，当前流程会做短距离反向卸载：

```text
unload_target = current_stroke - unload_mm
```

默认 `unload_mm = 0.05 mm`。到达目标或超时后电机失能，进入 `ClampDoneDisabled`。

该策略的目的不是松开工件，而是释放丝杆螺母的轴向预紧尖峰，降低抱死风险。实际卸载量需要样机标定。

### 5.5 打开

`Open` 当前流程：

```text
ClampDoneDisabled/Ready
  -> UnlockBeforeOpen
  -> Opening
  -> Ready
```

打开前先用低电流、小速度反向解锁，确认开始运动后进入正常打开。到达开爪端后重新设置软件零点。

## 6. 已解决的问题

| 问题 | 处理结果 |
|---|---|
| `python-can` 缺少 `slcan/pcan` hidden import | CMake/PyInstaller 打包加入 hidden import |
| 旧 SDK 与设备不匹配 | 改用 `assets/dm-tools-master/DM_DeviceSDK` 的 v1.1 路径 |
| `dmcan_find_devices_with_type()` 不稳定 | 使用全设备发现 `dmcan_find_devices()` |
| `dmcan_context_destroy()` 可能导致进程崩溃 | 关闭通道和设备后不主动销毁上下文 |
| UI 点击按钮没有 TX/RX | 修正按钮绑定，改为调用当前 controller/backend |
| Runtime Log 无法停在某行查看 | 增加自动滚动开关和选中行详情 |
| `Home low-current` 使能但不运动 | 修正未归零目标、首次模式切换、CANFD 控制帧默认值 |
| `Clamp force` 类似无运动 | 同步使用修正后的 POS_FORCE 运动命令 |

## 7. 当前遗留风险

后续不能忽略以下风险：

- 夹紧力映射未标定，当前不是闭环力控。
- 单圈绝对值编码器和 16 mm 丝杆行程之间的位置连续性、绕圈处理需要全行程验证。
- `Stroke sign` 必须在实机上确认，方向反了会导致回零和夹紧相反。
- 失能后的夹持保持依赖丝杆自锁，需要在不同线缆直径、表面材料、振动条件下验证。
- 卸载距离 `unload_mm` 需要通过夹持保持力和解锁成功率共同标定。
- 堵转检测阈值依赖摩擦和工况，必须做样机数据统计。
- 当前 UI 真实硬件实现集中在一个 Python 文件中，模块边界不适合长期维护。
- C++ 控制骨架尚未完全同步 Python UI 中最新的硬件联调修正。
- 达妙模式切换当前未严格等待寄存器回读确认，最终产品应加入确认和超时处理。
- 异常恢复策略目前较简化，防卡死分级动作还未完整落地。

## 8. 当前建议测试顺序

每次换硬件、换固件或重启设备后建议按以下顺序测试：

1. 打开 UI，选择 `Damiao USB2FDCAN`。
2. 确认 `Adapter FDCAN/BRS`、`Motor frames CANFD`、`Command ID + mode` 都已勾选。
3. 确认电机 ID、反馈 ID、通道和波特率。
4. 点击 `Connect backend`。
5. 点击 `CAN probe`，确认有 RX。
6. 点击 `Enable`，确认有 `TX FD id=0x7FF` 模式切换和 `TX FD id=0x308` 使能帧。
7. 点击 `Home low-current`，观察方向和电流。
8. 如果方向反了，立即 `Stop + Disable`，修改 `Stroke sign` 后重新连接。
9. 空载或低风险工况下测试 `Clamp force`。
10. 观察夹紧后是否执行卸载和失能。
11. 测试 `Open` 是否能低电流解锁并回到开爪零点。
